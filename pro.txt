# GeomModel CAD - Система геометрического моделирования

## Описание проекта

GeomModel CAD — это система автоматизированного проектирования (CAD) для создания и редактирования 2D геометрических чертежей. Проект реализует функциональность для работы с геометрическими примитивами, стилями линий по ГОСТ 2.303-68, навигацией и привязками.

### Реализованные лабораторные работы

- **ЛР №1:** Отрезок как базовый элемент (декартовы и полярные координаты)
- **ЛР №2:** Навигация (панорамирование, масштабирование, поворот)
- **ЛР №3:** Стили линий по ГОСТ 2.303-68
- **ЛР №4:** Геометрические примитивы и привязки

pip install -r requirements.txt

## Управление

### Управление мышью

- **Левая кнопка мыши:** рисование/выделение
- **Средняя кнопка мыши (СКМ):** панорамирование
- **Колесико мыши:** масштабирование
- **Правая кнопка мыши:** отмена операции

### Горячие клавиши

#### Инструменты
- `V` - Выделение
- `H` - Рука (панорамирование)
- `L` - Отрезок
- `C` - Окружность
- `A` - Дуга
- `R` - Прямоугольник
- `E` - Эллипс
- `P` - Многоугольник
- `S` - Сплайн

#### Навигация
- `+` / `-` - Масштабирование
- `[` / `]` - Поворот вида
- `F` - Показать всё
- `Home` - Сбросить вид

#### Редактирование
- `Delete` - Удалить выделенное
- `Escape` - Отмена операции
- `Tab` - Переключение режимов построения (для окружностей, дуг, прямоугольников, многоугольников)

## Архитектура

### Основные компоненты

#### 1. CADApplication (`app/application.py`)

Главный класс приложения, который инициализирует окно, создает UI компоненты и управляет инструментами.

**Основные методы:**
- `run()` - запуск главного цикла приложения
- `_on_tool_selected()` - обработка выбора инструмента
- `_on_point_entered()` - обработка ввода координат

#### 2. CADCanvas (`app/canvas/canvas.py`)

Основной холст для рисования, который управляет примитивами, сеткой и отрисовкой.

**Основные возможности:**
- Добавление и удаление примитивов
- Управление выделением
- Интеграция с инструментами
- Отображение сетки
- Система координат (миллиметры)

#### 3. NavigationController (`app/canvas/navigation.py`)

Контроллер навигации для управления видом (панорамирование, масштабирование, поворот).

**Основные методы:**
- `pan()` - панорамирование
- `zoom()` - масштабирование
- `rotate()` - поворот вида
- `zoom_to_fit()` - показать все объекты

**Математические преобразования:**
- Панорамирование: `x' = x + dx, y' = y + dy`
- Масштабирование: `x' = x * s, y' = x * s`
- Поворот: `x' = x * cos(θ) - y * sin(θ), y' = x * sin(θ) + y * cos(θ)`

#### 4. Grid (`app/canvas/grid.py`)

Система координатной сетки. По умолчанию шаг сетки в миллиметрах остается постоянным при масштабировании.

**Параметры:**
- `step` - шаг сетки в миллиметрах (по умолчанию 10 мм)
- `visible` - видимость сетки
- `adaptive` - адаптивная сетка (по умолчанию выключена)

### Геометрические примитивы

Все примитивы наследуются от базового класса `Primitive` (`app/primitives/base.py`).

#### Базовый класс Primitive

**Абстрактные методы:**
- `draw()` - отрисовка примитива
- `get_control_points()` - получение контрольных точек для редактирования
- `move_control_point()` - перемещение контрольной точки
- `get_snap_points()` - получение точек привязки
- `get_bounding_box()` - получение ограничивающего прямоугольника
- `contains_point()` - проверка попадания точки на примитив
- `get_properties()` - получение свойств для панели свойств
- `set_property()` - установка свойства

**Общие свойства:**
- `id` - уникальный идентификатор
- `style_id` - идентификатор стиля линии
- `selected` - состояние выделения
- `visible` - видимость
- `locked` - блокировка

#### Доступные примитивы

1. **Segment** (`app/primitives/segment.py`)
   - Отрезок по двум точкам
   - Координаты: `(x1, y1, x2, y2)`

2. **Circle** (`app/primitives/circle.py`)
   - Окружность
   - Координаты: `(center_x, center_y, radius)`

3. **Arc** (`app/primitives/arc.py`)
   - Дуга
   - Построение: три точки или центр + радиус + углы

4. **Rectangle** (`app/primitives/rectangle.py`)
   - Прямоугольник
   - Поддержка поворота и скругления углов
   - Координаты: `(x, y, width, height, corner_radius, rotation)`

5. **Ellipse** (`app/primitives/ellipse.py`)
   - Эллипс
   - Центр и две конечные точки осей

6. **Polygon** (`app/primitives/polygon.py`)
   - Многоугольник
   - Режимы: вписанный, описанный, обычный
   - Параметры: центр, радиус, количество сторон, тип

7. **Spline** (`app/primitives/spline.py`)
   - Сплайн по контрольным точкам

5. **ArcTool** (`app/tools/draw_tools.py`)

   - Центр и две конечные точки осей

8. **PolygonTool** (`app/tools/draw_tools.py`)
   - Режимы: вписанный, описанный, обычный (Tab для переключения)

9. **SplineTool** (`app/tools/draw_tools.py`)
   - Построение сплайна по контрольным точкам

### Система стилей линий

#### LineStyle (`app/styles/line_style.py`)

Определяет стиль линии согласно ГОСТ 2.303-68.

**Свойства:**
- `name` - название стиля
- `line_type` - тип линии (enum)
- `thickness` - толщина в миллиметрах
- `color` - цвет линии
- `dash_length` - длина штриха (мм)
- `gap_length` - расстояние между штрихами (мм)

**Типы линий (LineType):**
- `SOLID_MAIN` - Сплошная основная (толстая)
- `SOLID_THIN` - Сплошная тонкая
- `SOLID_WAVY` - Сплошная волнистая
- `DASHED` - Штриховая
- `DASH_DOT_THIN` - Штрихпунктирная тонкая
- `DASH_DOT_THICK` - Штрихпунктирная утолщенная
- `DASH_DOT_DOT` - Штрихпунктирная с двумя точками
- `SOLID_THIN_ZIGZAG` - Сплошная тонкая с изломами

#### StyleManager (`app/styles/style_manager.py`)

Менеджер стилей (Singleton), который управляет всеми стилями в приложении.

**Основные методы:**
- `get_style()` - получение стиля по ID
- `get_all_styles()` - получение всех стилей
- `get_current_style()` - получение текущего стиля
- `set_current_style()` - установка текущего стиля
- `add_style()` - добавление пользовательского стиля
- `update_style()` - обновление стиля

### Система привязок (Snaps)

#### SnapManager (`app/snaps/snap_manager.py`)

Система объектных привязок для точного позиционирования.

**Типы привязок (SnapType):**
- `ENDPOINT` - Конец линии/дуги
- `MIDPOINT` - Середина линии
- `CENTER` - Центр окружности/дуги/эллипса
- `INTERSECTION` - Пересечение примитивов
- `PERPENDICULAR` - Перпендикуляр
- `TANGENT` - Касательная
- `QUADRANT` - Квадрант (для окружностей)

**Основные методы:**
- `find_snap()` - поиск ближайшей точки привязки
- `enable_snap()` / `disable_snap()` - включение/выключение типа привязки
- `draw_markers()` - отрисовка маркеров привязки

**Параметры:**
- `snap_tolerance` - допуск привязки в пикселях (по умолчанию 10)

## Система координат

### Единицы измерения

Все координаты в системе используют **миллиметры** как базовую единицу измерения.

### Системы координат

1. **Мировая система координат (World Coordinates)**
   - Единицы: миллиметры
   - Начало координат в центре холста (по умолчанию)

2. **Экранная система координат (Screen Coordinates)**
   - Единицы: пиксели
   - Начало координат в верхнем левом углу окна

### Преобразования

Все преобразования выполняются через класс `Transform`, который хранит матрицу преобразования 2x3:

```
[ a  b  tx ]
[ c  d  ty ]
```

Где:
- `a, d` - масштабирование по X и Y
- `b, c` - компоненты поворота
- `tx, ty` - смещение (translation)

## Расширение функциональности

### Добавление нового примитива

1. Создайте новый класс, наследующий от `Primitive`
2. Реализуйте все абстрактные методы
3. Зарегистрируйте примитив в `PrimitiveFactory` (опционально)
4. Создайте соответствующий инструмент, наследующий от `BaseTool`
5. Добавьте инструмент в `CADApplication._tools`

### Добавление нового инструмента

1. Создайте класс, наследующий от `BaseTool`
2. Реализуйте методы `get_name()` и `get_icon()`
3. Реализуйте обработчики событий (`on_left_click`, `on_mouse_move`, и т.д.)
4. Добавьте инструмент в `CADApplication._tools` и `Toolbar`

### Добавление нового стиля линии

```python
from app.styles.style_style import LineStyle, LineType
from app.styles.style_manager import StyleManager

style_manager = StyleManager()
new_style = LineStyle(
    name="Мой стиль",
    line_type=LineType.DASHED,
    thickness=1.0,
    dash_length=10.0,
    gap_length=5.0
)
style_manager.add_style("my_custom_style", new_style)
```

## Технические детали

### Используемые технологии

- **Python 3.7+** - основной язык программирования
- **tkinter** - библиотека для графического интерфейса
- **ttk** - современные виджеты tkinter

### Архитектурные паттерны

- **Singleton** - `StyleManager` реализован как singleton
- **Factory** - `PrimitiveFactory` для создания примитивов
- **Observer** - система уведомлений об изменениях стилей
- **Strategy** - различные способы построения примитивов (режимы инструментов)

### Производительность

- Все примитивы кешируют свои canvas item IDs
- Перерисовка происходит только при необходимости
- Оптимизированные алгоритмы поиска привязок

## Известные ограничения

1. Работа только с 2D геометрией
2. Экспорт/импорт файлов не реализован
3. Нет поддержки слоев (layers)
4. Нет истории действий (undo/redo)
